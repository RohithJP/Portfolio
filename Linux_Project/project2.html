L2 — Ubuntu Server Hosting & Deployment

Deployment and troubleshooting of web applications on Ubuntu: NGINX, Node.js, systemctl services, firewall rules, ports, and server reliability.

Project Overview

This project covers deploying a Node.js web application on Ubuntu, configuring NGINX reverse proxy, diagnosing port conflicts, fixing apt cache lock issues, and performing service-level monitoring & debugging using systemctl and logs.

Deployment Tasks

  • Host Node.js app with reverse proxy using NGINX
  • Create systemd service file for auto-start
  • Fix “waiting for cache lock” apt issues
  • Resolve port 3000/8080 conflicts
  • Enable UFW and configure firewall
  • Debug service failures using journalctl
  • Check running ports using ss/lsof

Troubleshooting

  • Service not starting after reboot
  • NGINX failing due to bad config
  • Port already in use (PID conflict)
  • curl showing backend unreachable
  • Failed to bind server on port issue

Achievements

  • Production-style hosting experience on Ubuntu
  • Reverse proxy + service automation
  • Debugged all port and config issues

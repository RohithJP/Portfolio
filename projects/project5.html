Architecture Workflow

Detailed Explanation

This architecture demonstrates a fully deployed dynamic PHP web application running on EC2 with NGINX + PHP-FPM and a secure, production-grade MySQL RDS backend.

  • GitHub Repository: The PHP app is version-controlled and cloned to the EC2 instance.
  • EC2 Web Server: NGINX and PHP-FPM handle routing and dynamic content processing. Permissions, sockets, and virtual hosts are configured to avoid 403/502/500 issues.
  • PHP Application Layer: Contains the main logic, website content, and database connection files.
  • MySQL RDS Instance: A fully managed DB with secured inbound rules allowing access only from the EC2 security group. SQL schema imported during setup.
  • End Users: Access the website via public IP / domain, while backend operations stay private.
  • Security Focus: SG referencing, principle of least privilege, no DB exposure, PHP-FPM tuning, and MySQL optimizations.

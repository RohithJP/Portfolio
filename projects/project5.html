05 — PHP Website Deployment with MySQL RDS

End-to-end deployment of a PHP application using NGINX, PHP-FPM, and Amazon RDS. Includes debugging, permissions, and connectivity fixes.

Architecture

GitHub → EC2 (NGINX + PHP-FPM) → RDS MySQL → Users

• NGINX serves PHP via PHP-FPM
• RDS provides secure MySQL storage
• Application pulled from GitHub
• Permissions fixed for uploads
• PHP app connects to RDS

NGINX PHP-FPM RDS EC2 GitHub
(SVG remains unchanged)

Project Overview

Deployment of a PHP-based web application on EC2 with MySQL hosted on RDS.

Setup & Configuration

01
Launch EC2
Install NGINX, PHP, PHP-FPM.
02
Install PHP modules
sudo apt install php php-fpm php-mysql -y
03
Clone Website
git clone <repo>
04
Setup RDS
Create DB + user + import SQL dump.

Troubleshooting

  • 502 Bad Gateway — PHP-FPM socket mismatch.
  • 500 Internal Error — Missing PHP modules.
  • RDS Access Denied — Wrong SG rules.
  • Default NGINX page — default site not removed.

Techniques Learned

  • NGINX virtual hosts
  • PHP-FPM socket debugging
  • Linux permissions for www-data
  • Importing DB dumps to RDS

Achievements

  • Successful PHP + MySQL deployment
  • Resolved 502/500 debugging issues
  • Connected securely to RDS

Project Summary

PHP + NGINX + RDS deployment with real debugging and fixes.

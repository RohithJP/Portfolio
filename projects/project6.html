06 — PHP Website Deployment with MySQL RDS

Complete deployment of a dynamic PHP web application on AWS EC2 using NGINX, PHP-FPM, and Amazon RDS. Includes debugging for 502/500 errors, DB connectivity fixes, and Linux permissions.

Architecture

GitHub → EC2 (NGINX + PHP-FPM) → RDS MySQL → Users

• NGINX + PHP-FPM handles web requests
• RDS provides scalable MySQL database
• Website cloned from GitHub
• Permissions fixed to prevent 403/500 errors
• Full stack connected and validated

Project Overview

Deploying a PHP-based college website using EC2 + NGINX + PHP-FPM and connecting it securely to Amazon RDS.

Setup & Configuration

01
Launch EC2
Install NGINX and required packages.
02
Install PHP & PHP-FPM
sudo apt install php php-fpm php-mysql -y
03
Clone Website
git clone <repo>
04
Setup RDS
Create DB + import SQL dump.

Troubleshooting

  • 502 Bad Gateway — PHP-FPM service mismatch.
  • 500 Error — Incorrect PHP modules.
  • RDS connection issues — SG inbound rules.
  • Default NGINX page — default site not removed.

Techniques Learned

  • Linux permissions (www-data ownership)
  • NGINX virtual hosts
  • Debugging PHP-FPM
  • Importing DB dumps to RDS

Achievements

  • PHP application deployed successfully
  • Secure DB connectivity to RDS
  • Debugged multiple NGINX + PHP issues

Project Summary

Fully deployed PHP application on AWS using RDS + EC2.

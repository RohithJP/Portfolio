03 — Centralized Log Management (Rsyslog + CloudWatch)

Implemented secure, centralized log collection across multiple EC2 instances using Rsyslog, directory templating, and AWS CloudWatch integration.

View Repository →

Overview

Centralized system and application logs from multiple EC2 servers into a single Rsyslog log collector instance. Logs were streamed into CloudWatch for real-time visibility, alerting, and auditing.

Workflow Diagram

AWS Web EC2 → AWS DB EC2 → → Rsyslog Log Collector (EC2) → → CloudWatch Log Group → → Metrics & Alarms

Tech Stack

  • EC2 (Ubuntu 22.04)
  • Rsyslog TCP/UDP
  • CloudWatch Agent
  • Security Groups / VPC Networking
  • Linux log forwarding

Setup & Implementation

1. Log Collector EC2 Setup

  • Created /var/log/remote folder with syslog ownership
  • Enabled Rsyslog TCP/UDP listeners (port 514)
  • Implemented hostname-based directory templates
  • Restarted and enabled Rsyslog service

2. Web & DB Server Setup

  • Forwarded all logs using TCP over private VPC
  • Configured secure Rsyslog forwarding rules
  • Restarted Rsyslog on all servers

3. Testing & Verification

  • Validated forwarding via logger -p local0.info
  • Verified auto-creation of hostname log folders
  • Checked CloudWatch log groups for real-time logs

Troubleshooting

  • Fixed Rsyslog permissions preventing log creation
  • Resolved UDP packet drops via Security Group changes
  • Corrected template syntax errors
  • Used journalctl -u rsyslog -f for live debugging
  • Ensured CloudWatch Agent mapped correct log file paths

What I Achieved

  • Centralized Linux logs from multiple EC2 servers
  • Real-time CloudWatch monitoring & alarms
  • Structured hostname-based log storage
  • Improved auditability and troubleshooting
  • Hardened pipeline security with VPC internal routing

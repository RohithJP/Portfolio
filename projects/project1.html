01 — Multi-Tier WordPress Architecture

Nginx → Apache → MySQL deployed across public + private subnets with secure routing and hardened access.

Architecture Workflow

Detailed Explanation

This project demonstrates a production-ready 3-tier WordPress setup separating the reverse proxy, application server, and database to ensure scalability, security, and performance.

  • Public Nginx Tier: Handles inbound HTTP traffic, performs SSL termination, and forwards internal requests to Apache.
  • Private Apache Tier: Serves WordPress securely with no exposure to the internet. Protected using strict Security Groups and NACLs.
  • Private Database Tier: MySQL is isolated with access only from the Apache server. No external connectivity allowed.
  • Traffic Flow: All user requests → Nginx → Apache → MySQL, ensuring complete tier isolation and secure internal routing.
  • Monitoring + Hardening: CloudWatch logs, least-privilege SG rules, and WordPress security best practices applied.

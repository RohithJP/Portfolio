01 — Multi-Tier WordPress Deployment

A complete 3-tier WordPress infrastructure using NGINX reverse proxy, Apache/PHP web tier, and MySQL database tier — deployed with strict networking, SG rules, backups, and security hardening.

Workflow

Project Overview

This project delivers a production-style multi-tier WordPress application across isolated subnets, hardened network perimeter, automated backups, and fully documented operational workflow.

Goal

Deploy a secure 3-tier WordPress infrastructure using AWS EC2, following real-world reliability, scalability, and security best practices.

Architecture

  • Public Subnet (10.0.1.0/24) → Reverse Proxy
  • Private Subnet (10.0.2.0/24) → Web Tier
  • Private Subnet (10.0.3.0/24) → Database Tier

VPC & AWS Console Setup

Configured networking, NAT, routing tables, and security groups following best practices.

Troubleshooting

  • Private subnet SSH issues → Fixed using ProxyJump chain
  • MySQL host access denied → Corrected user@host patterns
  • Apache directory permissions → Ownership fixes for www-data
  • Nginx reverse proxy misrouting → Corrected upstream backend definition

Techniques Learned

  • Full VPC design (public/private subnets, NAT, routing)
  • Reverse proxying WordPress through NGINX
  • MySQL DB hardening and remote access configuration
  • Secure multi-tier EC2 SSH access using ProxyJump
  • Automation via backup scripts & user management

Achievements

  • Fully deployed 3-tier production-style WordPress stack
  • Proper network isolation and hardened security groups
  • Backup automation + user/role-based access
  • Application accessible through NGINX reverse proxy

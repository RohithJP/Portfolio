01 — Multi-Tier WordPress Deployment

A complete 3-tier WordPress infrastructure using NGINX reverse proxy, Apache/PHP web tier, and MySQL database tier — deployed with strict networking, SG rules, backups, and security hardening.

Workflow

Project Overview

This project delivers a production-grade 3-tier WordPress architecture using EC2-based reverse proxy, Apache/PHP web tier, and a dedicated MySQL database tier. The infrastructure was deployed manually to demonstrate AWS VPC design, routing, SG isolation, backups, and operational troubleshooting.

Goal

Deploy a secure 3-tier WordPress infrastructure on AWS EC2 following real-world reliability, scalability, segmentation, and security best practices.

Architecture

  • Public Subnet (10.0.1.0/24) — Reverse Proxy
  • Private Subnet (10.0.2.0/24) — Web Server
  • Private Subnet (10.0.3.0/24) — Database Server

VPC & AWS Console Setup

All VPC, subnet, routing, NAT, SG configuration steps remain documented exactly as you described in the project requirements. ✔ VPC ✔ Subnets ✔ IGW ✔ NAT ✔ Route Tables ✔ Security Groups

Major Troubleshooting

  • Fixed SSH access failures caused by missing public IP.
  • Resolved key mismatch + AMI boot issues.
  • Fixed MySQL remote access permissions.
  • Corrected Apache rewrite + VirtualHost settings.
  • Nginx proxy headers corrected to prevent redirect loops.

Techniques Learned

  • Multi-tier networking with security group references.
  • Reverse proxy load redirection & header management.
  • Linux system debugging (Apache, PHP, MySQL).
  • Backup automation and cron-based retention.
  • SSH ProxyJump and key-based secure access.

Achievements

  • Fully functional 3-tier WordPress stack.
  • Network segmentation and hardened access.
  • Automated MySQL backup system created.
  • Documented troubleshooting runbook.
  • Successfully deployed without managed services.

01 — Multi-Tier WordPress Architecture

Nginx → Apache → MySQL deployed across public + private subnets with secure routing and hardened access.

Overview

A 3-tier WordPress deployment inside a production-grade VPC. The setup includes: a public reverse proxy, private web server, private database tier, strict NACLs, monitoring, and secure SSH access via a bastion.

Architecture Summary

  • Nginx reverse proxy in public subnet
  • Apache + WordPress in private subnet
  • MySQL database (RDS-like but manual EC2 deployment)
  • Strict NACLs + Security Groups for tier isolation
  • CloudWatch monitoring enabled for all tiers
  • Automated provisioning using User-Data

Setup & Deployment Workflow

  • Create VPC → Subnets → Route Tables → Internet Gateway → NAT
  • Launch Nginx EC2 (public)
  • Launch Apache EC2 with WordPress (private)
  • Deploy MySQL DB EC2 (private)
  • Configure Nginx reverse proxy → Apache internal IP
  • Apply SG & NACL rules for strict tier isolation
  • Enable CloudWatch logs + metrics

Techniques Learned

  • Reverse proxy routing and SSL termination
  • Tiered VPC network design
  • Advanced Security Groups & NACL logic
  • WordPress hardening in private networks
  • EC2 troubleshooting & service debugging
  • Integration testing between Nginx → Apache → MySQL

Troubleshooting Highlights

  • Fixing 502/504 errors when proxying Apache
  • Resolving DB connection failures due to SG misconfiguration
  • Permission issues in WordPress uploads
  • MySQL bind-address and firewall hardening

What Was Achieved

  • Fully segmented 3-tier architecture with production security
  • Scalable WordPress deployment with reverse proxy
  • Private MySQL with no direct internet exposure
  • Monitoring, logs, and performance insights using CloudWatch

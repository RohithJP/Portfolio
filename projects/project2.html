02 — Automated EC2 Static Hosting

Fully automated EC2 provisioning using User Data + S3 sync. Ideal for fast static deployments & version-controlled website hosting.

Overview

This project deploys a lightweight static website hosting solution using EC2 and automated S3 syncing. A single EC2 instance boots, installs Nginx, pulls website files from S3, and continuously stays in sync using a cron-based automation loop.

Architecture Setup

  • EC2 instance running Ubuntu (t2.micro – free tier)
  • S3 bucket storing the static website source files
  • User Data script for auto-provisioning Nginx + AWS CLI
  • Cron job every 5 minutes for auto-sync updates
  • Security Groups allowing HTTP + SSH

Techniques Learned

  • Managing IAM roles for EC2 S3 access

User Data Script

#!/bin/bash
apt update -y
apt install nginx awscli -y

# pull website files
aws s3 sync s3://my-static-site-bucket /var/www/html/

# start nginx
systemctl enable nginx
systemctl start nginx

# cron auto-sync
echo "*/5 * * * * root aws s3 sync s3://my-static-site-bucket /var/www/html/" >> /etc/crontab
